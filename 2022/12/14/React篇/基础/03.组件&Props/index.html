函数组件与Class组件在事件处理上的一些区别

  • 使用class组件时的事件处理： onClick={() => this.props.onClick()}

  • 使用函数组件时的事件处理： onClick={props.onClick}（注意两侧都没有括号）。

提取原则

  • UI中多次被使用的部分

Props 的只读性（不能修改Props）

组件无论是使用函数声明还是通过 class 声明，都决不能修改自身的 props。

当然，应用程序的 UI 是动态的，并会伴随着时间的推移而变化。在下一章节中，我们将介绍一种新的概念，称之为 “state”。在不违反上述规则的情况下，state 允许 React 组件随用户操作、网络响应或者其他变化而动态更改输出内容
